iPulse について iPulse は Mac OS X 内部の動作をグラフィック表示するアプリケーションです。

iPulse を起動後、ゲージが幾つか iPulse ウインドウと Dock アイコン内に表示されます。これらのゲージは以下の項目を示しています；

  • CPU 使用状況
  • システム状況
  • メモリ使用状況 と スワップ
  • ディスク 使用状況 と 読込/書込状況
  • ネットワーク稼働状況
  • モバイル (バッテリー＆ワイアレス)
  • 時計 (現在時刻と日付)
  • 一般情報
上記の名前部分をクリックすると、各ゲージと、ゲージ上にカーソルを当てると表示される情報ウインドウについての詳細説明にジャンプします。 イントロ では役に立つ背景情報なども説明しているので、できればそこから読み始める事をお勧めします。

iPulse は 環境設定—システムの情報を表示する様々な方法があります—で調整可能です。また iPulse ジャケットを読み込んで環境設定を素早く設定できます (以下の項目 ジャケットの作成 でその方法を説明しています)。

iPulse を使い慣れてきたら、 Tips , トリック と パワーユーザ の項目で iPulse 調整についての追加ヘルプをご覧ください。

注意: このページは iPulse の初回起動時に表示されます。もう一度見る必要がある場合には、 iPulse のメニューバーからヘルプを選択してください。

How It Works お気付きのように、Mac OS X は Unix OS を基礎としています。この OS は長年の内に、堅牢で能率的なプラットフォームへと進化しました。Mac OS X とその先代の開発者達は OS の内部状況をチェックする為のメカニズムを組み込んできました。この機能のおかげで、管理者はシステムの最適化とトラブルの原因特定を行えるのです。

iPulse の最終目標は、システム内部の情報を省スペースで分かり易くグラフィック表示する事です。

iPulse はシステム内部の情報を何処から入手しているのですか？ またそれをどのようにしてグラフィック表示しているのですか？この項目の残りの部分ではそれらのご質問に答える為に説明されています。

イントロ このヘルプをより幅広い層の人たちに役立てて頂けるように、各項目では基礎レベルから説明を始めています。説明を進めるにつれ、より専門的になっていきます—難しすぎると思ったら読み飛ばして次の項に進んでください。

iPulse のこのバージョンでの大きな新機能の一つは、ゲージ上をマウスオーバーすると表示される情報ウインドウです。このウインドウの内容についてはゲージについての説明の後で話題に上っています。多くの場合、情報ウインドウはそのゲージが表している内容を詳しく知る手助けになります。

情報ウインドウについての説明の後で、このアプリに技術的に興味がありデータを十分に理解したいという方に役立つような初歩的な情報を提示しています。ターミナルウインドウから実行可能な Unix シェルコマンドについて言及しています。興味のある方は先に進んでコマンドを試してみてください: ここで挙げている例はいずれもお使いの Mac に損害を与えるものはありません(コマンドの綴りだけは間違えないようにして下さい！)。特定のコマンドについて詳しく知りたい方は man [command] をお試しください。

CPU 使用状況 最初に説明するゲージは CPU: あなたの Mac と、iPulse の表示の心臓部です！

Mac OS X は Unix 式の CPU 分析をとっており、３つに分類されます: system time, user time と nice time です。 system time は OS 側のプロセスに費やされる時間です。 user time は一方で、カーネルの外部で実行されるアプリケーションとサービスに費やされる時間です。 nice time は低優先度プロセス — これらは優先度が低い為、システムがビジーな状態の時にカーネル側で自動的にその CPU 使用率を下げる事ができるので、 "nice" と呼ばれます — の実行に費やされる時間です。

CPU ゲージは system time の％を内側の円 [1] に、user time の％を外側の円 [2] に、nice プロセスを user の円の外側の円 [3] に表示します。ゲージに表示される数は system, user そして nice time の合計値です。この値は過去10回の更新時の値の平均値です。

CPU ゲージのもう一つの機能は システムによる負荷を色合いの濃淡で表示します。システム負荷については次項で説明しますが、電脳的な帯熱要素とでもお考えください：CPU が熱くなればなるほど、よりゲージの色は濃くなります。

iPulse はお使いの Mac が過去15分間にこなした仕事量を調査し、その結果の値をもとにゲージの色合いを調節します。その最終結果を CPU 使用状況の長期の傾向としてご覧頂けます。例えば、2GB のイメージ上でガウスのぼんやりとした明滅で表示されるような、 CPU の集中的なタスクを行っている場合、ゲージがどんどん赤く(熱くなっています！)なっていくのをご覧になるでしょう。ゲージが標準の色に戻った時、実行プロセスが終了したとわかるでしょう。

iPulse の各ゲージ上にカーソルを乗せた時、情報ウインドウが表示されます。これを見れば各ゲージがそれぞれどこに位置しているのかが分かりやすくなります。しばらく iPulse を使ってみれば、わざわざ思いだす必要もなくなるでしょうが、初めての時はそれぞれのゲージと表示している項目を関連づけるのはちょっと難しいでしょう。

CPU 情報ウインドウは以下の項目を表示しています :
  • 読み込み - システムによる現在の CPU 読み込み (1分前の平均値です) を、最小値、最大値と過去1時間の平均値に沿って表示します
  • プロセス - 現在有効なプロセス数を実行、スリープ、その他に分けて表示します。"root" プロセスは適切なアクセス権を持たないので iPulse では調査できません。iPulse のアクセス権を手動で変更すれば、より正確なプロセス情報を表示できます。詳細は パワーユーザ の項を御覧ください。
  • 温度 - CPU チップ上のセンサが報告した現在の CPU の温度です。この情報を得られない機種も幾つかあります (特に、最新の PowerMac G4 など) 。 CPU 温度が報告された場合でも、そこまで正確な訳ではありません : 実際の温度よりもその値は低くなりがちです。この値は相関値と考えるのが良いでしょう。 G4 チップの最高温度は 101 °C です — もしこの数値を目にしたら、一目散で逃げるしかありません！ :-)
  • 使用状況 - CPU 使用状況は上記の分類ごとにパーセンテージで表示されます。デフォルトでは、合計値は低優先度プロセスを含みません (環境設定で変更可能です)。
  • プロセスリスト - 最後の情報は平均 CPU 使用状況により分類されたプロセスリストです。最も負荷の高いアプリケーションがリストのトップに表示されます。プロセス番号 (プロセス id または procid とも呼ばれます) 我々 Unix 信者のために表示しています。

ターミナルウインドウで top コマンドを使用すればこれの類似情報を入手できます。実際、 iPulse が、標準より CPU time が増大しているといったような何か予期せぬ報告をした場合、 top コマンドを使用して問題を調査できます。情報ウインドウで表示されているプロセス番号も ps -p # ( # の位置にはプロセス番号を入力します) コマンドに使用できます。

CPU 使用状況は HOST_CPU_LOAD_INFO セレクタなどの host_statistics() システムコールを使って読み込まれます。

システム状況 UNIX 界での CPU 使用状況のもうひとつの計測方法が "load average" です。 load average 一定時間の間で待機中のプロセスの平均数をメートル法で示しています。 CPU の読込が増加すれば、 load average も増加します。

load average は銀行で並んでいる列のようなものとお考え頂けるでしょう。銀行員 (プロセッサ) と取引する必要のある顧客 (プロセス) です。列 (順番待ちプロセス) が長くなるにつれ銀行員の数も増え (マルチプロセッサ) 、銀行員のこなす仕事も増えます。お昼時にはサービス待ちの顧客がさらに増え、銀行員の load average は一日の他の時間帯でのそれよりも高くなります。

履歴ゲージは load average をドットの輪として iPulse ウインドウの外側に表示します [19] 。このゲージは過去1時間での読み込み量の概要を表示します。現在の分は空白のドットで表示されます ; 別のドットで現在の分に関連した load average を表示します。例えば、5つの空白のドットが時計の逆周りで表示されていれば、それは過去5分間の履歴を表示しています。

不透明なドットは 負荷の高かった分です (メールチェック、 Photoshop などでの負荷の高い作業もしくは CPU をかなり消費する作業) 。あるいは、透明なドットは CPU 負荷の低かった時間の履歴です。例えばコーヒーブレイク中、とか。

load average の表示は環境設定ウインドウ内の履歴パネルで調整可能です。最小 load average のスライダーで、表示される load average の最小値 (この値以下の場合は透明表示になります) を設定できます。最大 load average はドットが不透明表示に変わる時のレベルです。これらの環境設定は既出の CPU の濃淡表示にも使用されます。

履歴ゲージには情報ウインドウはありません。load average の統計は CPU ゲージの情報ウインドウで御覧になれます。

ターミナルウインドウから uptime と top コマンドを使用して現在の load average を表示できます。この場合の load average の履歴は 1, 5 から 15 分です。

iPulse は HOST_LOAD_INFO セレクタを使用した host_statistics() システムコールによる CPU 負荷情報を表示しています。

メモリ使用状況 Mac OS X は仮想メモリの管理に、とても洗練されたメカニズムを持っています。もし、今まで不幸にも OS 9 での仮想メモリを嫌というほど経験しているのでしたら、この洗練されたメカニズムに感謝しているでしょう！

Mac OS X での仮想メモリは物理メモリの使用状況を4つのカテゴリ : wired, 使用中 , 未使用, 空き, に分類して管理しています。これらのカテゴリはページと呼ばれる 4KB ごとのメモリブロックで計算されます。カテゴリについては以下に記述します:

  • wired [4] - Unix カーネルが使用しているページです。これらのページは "その場でロック" されていて物理メモリからの再読み込みはできません。カーネルが管理しているデータ（プロセスやスレッド、ライブラリ）は wired メモリに保存されているので、アプリケーションは wired メモリの使用に間接的に影響を及ぼします。
  • 使用中 [5] - これらのページは最近アクセスされた物理メモリの量を示します。使用中ページはアプリケーションにアクセス可能です。
  • 未使用 [6] - これらのページは最近アクセスされていないメモリです。未使用ページのデータは有効ですが、アプリケーションにはアクセスできません。最終的に、未使用ページは物理メモリ上からディスクへと移動されます (新規ページの為にスペースを空ける為です)。
  • 空き [7] - これらのページはシステムのどの部分にも使用されておらず、すぐにでも使用可能です。
Mac OS X にはそれらカテゴリの状態を定期的にチェックして一方から他方へメモリページを移動させる "pager" というものが幾つか存在します。このメカニズムの背景にある演算方法についてはこの書類の範囲では収まりません ; "VM pager" で Apple のデベロッパサイトを検索すればすべての詳細を御覧になれます。このサイトでは virtual memory サブシステムについてのより一般的な情報も掲載されています。

iPulse がメモリ使用状況をグラフ表示する際、上記の4カテゴリを2色の帯で表示される "使用中" と "使用可能" に分けて表示します。"使用中" の色は wired の量 (左端) と使用中メモリです。"使用可能" メモリは未使用と空きメモリ (右端) を別の色で表示しています。

ゲージ上にカーソルを乗せた時、どのカテゴリのメモリを指しているのか、どれだけのスペースが割り当てられているのかをインジケータがハイライト表示します。合計の後に、最もメモリを使用しているプロセス (アプリケーション) のリストを御覧になれます。リストでは使用中 (resident usage) の物理メモリ量を数値と合計メモリのパーセンテージの両方で表示しています。アプリケーションで使用中のメモリ (仮想メモリ) の合計量も御覧になれます。

再び、ターミナルウインドウ内で top コマンドを使用すればこの情報が表示され、メモリが各プロセスにどのように配分されているのかの詳細もわかります。 vm_stat コマンドでは仮想メモリがどのように使用されているのかの詳細もわかります。

iPulse は Unix カーネルからメモリ情報を収集する際に HOST_VM_INFO セレクタを使用して host_statistics() を呼び出しています。

スワップメモリ スワップメモリ (または "paging") は物理メモリからハードディスクへとメモリページを移動させる作業の事です。この作業にはかなりのシステムリソースを使用します : メモリを読み込みディスクへコピーされます。スワップ中の Mac は、他の重要なタスク (ゲームでのトーナメント戦など) をこなす十分な時間はありません。

スワップイン (pagein とも呼ばれます) はアプリケーションがメモリページにアクセスする必要がある場合で、そのページが使用中でない場合に発生します。ページはディスク上から読み込まれなければなりません。同様に、新たなメモリの要求にスペースを空ける為、未使用ページは スワップアウト (pageout) されてディスク上に書き込まれます。

Mac OS X をお使いならこの現象にお気づきでしょう : たくさんのアプリケーションを起動していてメモリが一杯になった時、アプリケーションの切り替え時の遅れとディスク音にお気づきでしょう。これは新規アプリケーション用にメモリページがスワップされ、一方で古いアプリケーションからのページがディスクにスワップアウトされる事から起こるものです。

メモリのスワップがあまりに頻繁に発生する場合は、お使いの Mac にメモリを増設しろというサインです！

スワップイン/アウトページ用のゲージは1から最大90まで増加するページ数を表示しています (ゲージの弧の段階につき1ページです)。スワップインページは左側 [16] に、スワップアウトページは右側 [17] に表示されます。グラフ上のテキストはスワップページの数を示します。

スワップゲージ上にカーソルを乗せた時、ページ状況の合計を表示した情報ウインドウが御覧になれます。これはディスク上にメモリを蓄積するのに使用されるファイル数も報告しています ; "swapfile" の使用があまりに多いようでしたらそれは仮想メモリが断片化しているので、再起動すれば整理されるでしょう。

プロセスリストは2つの重要な情報によって分類されます :

  • Pageins - ディスクからメモリへ読み込む為にプロセスが使用したページの数です。
  • Faults - アクセスの必要はあるが有効ではないページの数です。実際にどちらのメモリをアプリケーションが使用しているのかを示しています。fault カウント数が多いから必ずしも悪いというわけではありません。

iPulse は Unix カーネルからメモリ情報を収集する際に HOST_VM_INFO セレクタを使用して host_statistics() を呼び出しています。

ディスク使用状況 iPulse はディスク使用状況ゲージを表示して、どれだけのスペースを使用しているのか一目ですぐ分かるようにしています。 iMovie, iPhoto や iTunes などのアプリケーションをお使いの方は大抵、空きスペース表示は短いでしょう！

ゲージ内の乱れを最小限に抑える為、読込/書込可能なディスク (ファイルシステム) のみを表示しています。CD-ROMs やその他読み込み専用メディアは表示されません。ネットワークディスクはサーバ接続語のみゲージ内に表示されます。

ディスク使用スペース量 [10] と空きスペース量 [9] はどちらも時計回りで表示されます。使用スペース量が 90% を超えると、ゲージは警告色で表示されます。iPulse で表示可能な使用ディスクは接続している最初の６つまでに制限されます。リスト内の最初のディスクは Mac OS X の読み込みに使用されているので Unix 標準方式に従って "(Root)" と名付けられます。

バージョン 2.0 では、全ディスクの合計を表示する新機能が追加されました — iPulse は全ディスクで使用されているスペース量を算出しその結果を表示します。ディスクを複数お持ちの方は、個別ににそれを表示させるよりはこれは結構便利な機能です。

ディスク使用状況ゲージ上にカーソルを乗せると、各ディスクとその使用スペース状況の合計が御覧になれます。リストはまた、ディスクで使用されているファイルシステムのフォーマットも表示しています。ファイルシステムのフォーマットは以下のどれか一つです :

  • hfs - 階層ファイルシステム。Mac の伝統的ファイルシステムです。
  • webdav - Web ベースのオーサリングとバージョンのファイルシステム。Apple が iDisk の替りにこう呼ぶようにしたのは我々には嬉しいです！
  • smbfs - サーバメッセージブロックファイルシステム。Windows でのネットワークファイル共有時に使用されます。
  • ufs - Unix ファイルシステム。1 MB ディスクでボリュームが十分だった黎明期の頃から使用されています。
  • msdos - MS-DOS ファイルシステム。すべてのフロッピーに必要です！
  • nfs - ネットワークファイルシステム。Unix と Linux での一般的なファイル共有メカニズムです。
  • udf - ユニバーサルディスクフォーマットファイルシステム。CD と DVD 録音に使用されます。

ターミナルウインドウから df -k コマンドを使用すればマウントされたすべてのファイルシステムの使用状況を表示できます (読み込み専用とデバイスファイルシステムも含みます)。

ディスク統計情報は getmntinfo() コールによって収拾されています。.

ディスク読込/書込状況 上記のディスク使用状況ゲージはほとんど静止状態のディスクの使用状況を表示しています。一方で動的な使用ゲージは、システムにディスクがどのように使用されているのかを表示します。ゲージは Unix カーネルによるすべての読込/書込作業をインジケータで表示します。また転送データ量も表示しています。

読み込み状況はゲージの左側 [8] に、書き込み状況はゲージの右側 [11] にドット表示されます。ドット表示は、最新の更新期間内での読込/書込作業を表示しています。100 以上の作業が行われた場合、ドットは高レベル状況を示すハイライト色で表示されます。

ディスクから、またはディスクへの転送バイト量は読込/書込状況インジケータの下に棒グラフ表示されます。グラフの表示単位は対数または自動に設定可能です — これらの機能については 環境設定 項目で説明しています。

棒グラフはまたピーク値も表示しています。iPulse は一定期間内の最大バイト数を維持し、ピーク値を細い線で表示します。初めは、ピーク値は1分ごとに維持されます ; この時間は一般環境設定で調節可能です。

ゲージ上にカーソルを乗せた時、読込/書込作業の合計が表示されます。この数値は作業の数と最新更新期間内での1秒毎の転送バイト数を表示します。現在のピーク値と平均値もまた表示されます。

ディスク読込/書込情報は"IOBlockStorageDriver" セレクタを使用して IOKit から入手しています。

ネットワーク状況 iPulse の設計時に頭を悩ませた問題の一つが、どの種類のネットワーク情報を表示すればよいかという事でした。カーネルからは数多くのネットワーク情報を入手可能です。結局、2種類の情報が最も役に立つということに決定しました :

  • IP 状況 - IP (インターネットプロトコル) による受信/送信パケット数です。これでどんな IP パケット (TCP パケットでなくても) の状況も御覧になれます。
  • TCP 帯域 - TCP (トランスミッションコントロールプロトコル) による送信/受信バイト数です。これでサーバ間でのファイル移動や、ウェブサーフィンや e-メールの受信などの一般的な作業時の転送データ量を御覧になれます。

iPulse は下りネットワークを左側 [12,13] に、上りネットワークを右側 [14,15] に表示します。棒グラフは1秒後とのデータバイト数 [13,14] を表示します。グラフの単位は 環境設定 項目に従って対数または自動に設定可能です。

IP パケット受信時には棒グラフ開始点 [12,15] にドットが表示されます。1秒間に25パケット以上の受信時にはドットがハイライト表示されます。

ゲージ上にカーソルを乗せると、ネットワーク回線状況の合計が表示されます。この数値には更新期間内での転送パケット数が含まれます。現在のピーク値と平均値も表示されます。

この情報ウインドウは幾つかの基本的なネットワーク設定情報 (インタフェース名とそれに関連した IP アドレス) も表示されます。

ターミナルウインドウ内で netstat -I [interface] コマンドとインタフェース名 (en0 が Ethernet で en1 が Airport など) を入力すればネットワーク情報を御覧になれます。追加ネットワーク設定情報は ifconfig -a -u コマンドで御覧になれます。

ネットワーク情報は sysctlbyname() システムコールを使用して入手しています。TCP 統計情報は "net.inet.tcp.stats" セレクタ、IP 統計情報は "net.inet.ip.stats" セレクタを使用しています。もし、これだけたくさんの情報を今言った作業だけで入手していると信じていただけない場合は、 sysctl -A net をお試しください。

モバイル バッテリーまたはワイアレス Ethernet (AirPort) を Mac でお使いの方は、幾つか追加ゲージを御覧になれます。ひとつは現在のバッテリー電力レベル [18]、もう一つはワイアレス信号強度 [20] です。

バッテリーゲージはフル充電時に6つのドットで、空の時にはドット無しで表示されます。バッテリーレベルが10％以下になると背景がハイライト表示されます (デフォルトでは赤です。あなたのお仕事のデータを救う、目に見える手がかりになります！)

バッテリーが充電されるにつれ、ドットはオレンジで表示されていきます。フル充電されるとドットは緑に変わります。このカラーリングは標準電力アダプタに合わせてありますが、違う色に変えたい場合はモバイル環境設定パネルで変更可能です。

ワイアレス信号強度は同様の方式で表示されます。.最大強度は6つのドットで、信号無しの時はドット無しで表示されます。信号の強度が10%以下になると、背景がハイライト表示されます — カフェの店内で新しい席を探す頃合いだという警告です！

これらのゲージの情報ウインドウはどちらもバッテリーとワイアレス接続についての追加情報を表示しています。バッテリーの電圧と現在の数値は残り時間の見積に沿って表示されます。ワイアレス情報はアクセスポイント名と ID を信号とノイズの比率に沿って含んでいます。

バッテリー情報は IOPMCopyBatteryInfo から、ワイアレス情報は書類化されていない Apple80211 framework から入手しています。

時計 時計は現在時刻を表示しています。

デフォルト環境設定では、データがどうやって表示されているのかは目に見えて明らかです。24時間周期の設定は時計の外観を変更し、その日の一時間一時間が独特の位置に表示されます。 24時間時計 はマシンの日付よりも前の日付になります — もともとは日時計に使用されていました。

時計はまた、日付も表示できます。環境設定で月日、曜日または月相を設定できます。月相は1年をおおよそ12の期間に区切ります。どうして12の数字が時間表示に頻繁に現れるのか気になっているなら、それは月のせいにしていいのですよ！

時報機能も設定可能です。独自のサウンドを追加する方法については Tips , トリック の項に説明されています。

日付部分 (12時間時計の位置で) の上にカーソルを乗せれば、時計情報ウインドウが表示されます。このウインドウには数値で時刻と日付 (タイムゾーンと GMT/UTC オフセットに沿っています) が表示されます。現在の曜日と日をハイライト表示したカレンダーの表示されます。

情報ウインドウには最後に Mac OS X を再起動した時からの稼働時間 (通常 uptime と呼ばれます) もまた表示されます。また、星座観測がお好きな方はユリウス暦と月相の詳細表示に感謝される事でしょう。

現在時刻は time() システムコールから入手しています。時刻は localtime() によって現地時間に変換されています。月相の算出は本来 John Walker の仕事に由来しています。— 彼のサイト Earth and Moon Viewer は特にお勧めです。

一般情報 最後に、しかし最小ではありません。iPulse ウインドウのゲージ無しの部分にカーソルを乗せればいくらかの一般情報が表示されます。

この情報ウインドウはコンピュータ名とあなたのログイン名 (正式名と略名の両方とも) を含んでいます。iPulse と Mac OS X バージョンもまた、iPulse の起動時間の後に表示されます。

最後に、一般情報にはマウスクリック判定の有無も表示されます。これで iPulse をクリックした時に実際に何か反応があるかどうか分かります :-)

環境設定 iPulse には数多くの環境設定があります。それらひとつひとつをねちねちと詳細に説明する代わりに、幾つかの主要な機能にハイライトをあてる事にします。まだあまり明らかではない設定にも焦点をあてていきます。

ジャケットの使用 環境設定項目を手っ取り早く変更するには iPulse ジャケットの読込を行います。iPulse 起動後、ジャケットファイル (拡張子が .ipulse のファイルです) をダブルクリックするだけでファイルの設定が適用されます。

ダウンロードした iPulse フォルダには幾つかのサンプルジャケットが含まれています。我々の オンラインギャラリー でも追加ジャケットを作成しています。このギャラリーは定期的に更新されます — このページをチェックするにはヘルプメニューからリンクしてください。

ジャケットに変更を加えたい場合は、コピーをとりましょう。メイン環境設定パネルを開いて設定の保存ボタンをクリックしてください。それからファイル名を指定し、OK ボタンをクリックすれば更新された設定で新規ジャケットを作成できます。

ゲージのオン/オフ チェックボックスをクリックすればすべてのゲージをオン/オフできます。これであなたのお好みに合うように iPulse の外観をすぐに整えられます。

透明度 iPulse のカラーウェルはすべて透明度をサポートしています。これでゲージの色の種類の調節だけでなく、色の強弱の調節も可能になります。

ゲージの透明度を調節するには、カラーウェルをクリックするだけです。カラー調節ウインドウ内で、不透明度スライダーを 0% (完全に透明) から 100% (完全に不透明) までドラッグできます。変化はすぐにゲージに反映されます。

ゲージの透明度を 0% に調節すると iPulse ゲージのパーツを簡単に隠せます。例えば、ウインドウの背景を 0% に設定してゲージをフローティングウインドウにすれば、ディスクとモバイル用背景にも使えます。同様に、使用可能メモリでなく使用中メモリ量を御覧になりたければ、使用可能メモリゲージの透明度を 0% に設定する事もできます。

ゲージ上のテキスト 大半のゲージにはテキスト表示機能があります。デフォルトでは、テキストはフローティングウインドウ上でのみ表示されます — 一般環境設定パネル内の設定で Dock アイコン上にもテキストを表示できます。

数値の単位 ディスクとネットワーク使用状況にはゲージの表示単位を選択できる環境設定があります。表示単位は :

  • 固定 - 単位はポップアップメニュー内で選択した項目の値で表示されます。例えば、もし "100KB/秒" を選択した場合、その秒における数値が 100KB の場合にはゲージは完全にフル状態になります。ゲージが半分の場合 (90° の弧の 45°) 数値は 50KB を示しています。
  • 自動 - この単位は上記の固定スケールとよく似ていますが、最大値を自動的に調整する点が異なります。調整は現在のピーク値を使用して行われます。例えば、現在のピーク値が 75KB/秒ならばゲージの単位は次の 10 の累乗 — 100KB/秒に設定されます。
  • 対数 - この単位はゲージを3つの項目に分類します。各項目は 10 の累乗の範囲でデータを表示します。例えば、もし対数の 10K 〜 1MB/秒を選択した場合、1K 〜 10K の値は始めの 30° 部分に表示されます。10K 〜 100K の間の値は次の 30° の弧に、100K 〜 1MB/秒の値は残りの 30° に表示されます。もし数値が 1KB/秒以下の場合は何も表示されません — 数値が 1MB/秒以上の場合はゲージは完全にフルになります。
さらに、単位が自動の場合はゲージ上に連続するドット数で現在の単位を表示します。ドット無しなら 1K, ドット1つで 10K, 2つで 100K, などなど。同様に、対数表示の場合は3つの範囲の間に薄い分離線を表示してどれだけのデータが表示されているのか分かるようにしています。

ピークインジケータ ピークインジケータはディスク使用状況とネットワーク状況ゲージで調整可能です。これらのインジケータはオーディオ録音装置におけるレベルメータのようなものです。

インジケータは一定期間内での最大数値のレベルを維持する事によって作動します。この最大値はゲージ上に薄い線で表示されます。

最大数値レベルの表示 (維持) 時間は環境設定の一般設定パネル内でコントロール可能です。もし表示単位を自動に設定している場合は、ピーク値の維持時間を調節すれば表示単位の調節頻度にも影響を及ぼします。

読込値の調整 履歴ゲージを表示している場合はおそらく、load average の最小/最大値を調節する必要が出てくるでしょう。デフォルトの値は幅広い範囲のマシンに適していますが、お使いのマシンごとにより正確な設定をする事も可能です — iBook または デュアルプロセッサ G5 用などに。

まず初めに皆さん、CPU ゲージ上にカーソルを乗せて情報ウインドウを表示させることでしょう。情報ウインドウの初めの部分に、"読み込み" という行内に最大値と最小値を御覧になれるでしょう。これらの値は環境設定の履歴パネルで設定した時の load average の値として最初に使用されます。

最小/最大値のスライダーを調節するにつれ履歴ゲージのドットの透明度が変化する点にお気づきでしょう。最小 load average では表示される最低 load average 値 (これ以下の値はすべて透明表示になります) を設定します。最大 load average はドットが不透明表示になるレベルです。

これらの環境設定は CPU ゲージの明滅の設定にも使用されます。最小読込み値では透明 (非表示) になります。最大読込み値では明滅は完全に不透明になります。

装飾 本バージョンの iPulse で人気の機能のひとつが、背景画像追加機能です。iPulse ゲージの背景にお好きな画像をどんなものでも追加できます。ゲージの透明度調節機能と組み合わせれば、iPulse の外観をほとんど自由に変更できます。

以下のジャケット作成の項目に背景画像のフォーマットとサイズについての情報を説明しているので御覧ください。

マウスクリックの無視 環境設定内のウインドウ設定パネルの幾つかの項目は明らかに分かりますが、そうでないものがひとつあります。 "マウスクリックの無視" です。

これは本バージョンの iPulse での新機能です。チェックをオンにすると、iPulse のフローティングウインドウ上でマウスをクリックしても無視されて、その下にあるアプリケーションがクリックされます。これで iPulse の下にあるスクロールバーやデスクトップやその他何でもクリック可能になります。

この機能の難点は、クリックが無効なので iPulse ウインドウを移動させるのが難しい点です。それは次の項に繋がります...

ホットキーの設定 iPulse では2つのホットキーを設定可能です。ひとつは iPulse フローティングウインドウの表示/非表示です。もうひとつは上記で話題に上った "マウスクリックの無視" 機能の切り替えです。

デフォルトでは、ホットキーは設定されていません。一般環境設定を開いて "設定..." ボタンを押してください。ウインドウが表示されるので割り当てたいキーコンボかファンクションキーを押して OK をクリックしてください。別のアプリケーションで使用されているキーコンボは割り当てないようにお気をつけください。コマンド-C をホットキーに割り当てた場合、クリップボードへのコピーではなく iPulse が起動します！

ホットキー設定後は、どのアプリケーションでも使用可能になります。表示/非表示ホットキーはプレゼンテーションやゲーム時に、また他のどんなフルスクリーンモードのアプリケーション起動時でも iPulse を隠すのに使えるので役立ちます。"マウスクリックの無視" 機能をお使いの場合に素早くマウスクリックを有効にでき、iPulse フローティングウインドウを移動させてその後でまたマウスクリックを無効にできます。

iPulse をリストから除外 最後に、一般環境設定パネルで iPulse をプロセスリストに含める項目があります。この機能は使用 しない 事をお勧めします。

その理由は、iPulse がどれだけ CPU を消費しているか知られたくないからではなく ; 自分自身をモニタしているプロセスとは周知の通り正確ではないからです。もし iPulse の消費時間の正確な見積を知りたければターミナルウインドウから top コマンドをご使用ください。

iPulse は CPU 消費量が多すぎるとお考えでしたら、Tips , トリック 項目にその消費量を下げる方法を説明しているので御覧ください。

ジャケットの作成 独自の iPulse ジャケットを作成して友人に配布したりご自分の Webサイトからダウンロードできるようにしたい場合の為に、ここに役立ちそうな方法を幾つか説明しておきます :

背景画像 iPulse ジャケットにおいて視覚的に最もインパクトのある部分がゲージの背景画像です。どんな TIFF, JPG, PNG, GIF, PDF もしくは ICNS ファイルでも、その他の環境設定パネル内で操作すれば背景に使用できます。TIFF 形式をお使いの場合、アルファチャンネルやレイヤーは使用しないでください。

画像ファイルの寸法は 128 x 128 ピクセル (ユーザが iPulse のウインドウサイズを変更した場合は自動的に縮尺されます) です。ゲージは 128 ピクセル — 8, 16, そして 32 ピクセルの等分割内に配置されます。サークルは時計と同じように — 30° 毎で 12 項目 に分割されます。

設定の保存 iPulse 設定をお好きなように調整したら、メイン環境設定パネルを開いて "設定の保存" をクリックしてジャケットを作成します。これで設定を含んだ .ipulse ファイル (ジャケット) が作成されます。

ジャケットは環境設定のうちの、ウインドウ設定と一般設定の項目を含まない点を御承知ください。これらに含まれるホットキー、ウインドウサイズや他の項目などは個人的な設定と考えられますので別の iPulse ユーザがジャケットを読み込んでも更新されません。

Tips , トリック この項ではお使いのシステム上で iPulse を調整する方法についていくつかご説明します。

ログイン時に iPulse を起動 システム環境設定のログイン項目を使用すれば自動的に iPulse を起動させるよう設定できます。

iPulse の CPU 消費率を下げる 旧型の Mac をお使いの方は、iPulse の CPU 使用率に関心がおありでしょう。iPulse をできるだけ能率的に起動させる方法をここに幾つか説明しておきます :

  • 更新間隔を長くする - 一般環境設定パネル内に更新間隔の設定項目があります。iPulse の主な CPU の用途は更新による必要な情報の収集ですので、速度をデフォルトの1秒から2秒 (もしくは3秒) に変更すれば消費率は極めて減少します。
  • iPulse の優先度を下げる - 一般環境設定パネル内の別項目に iPulse のプロセス優先度の設定があります。確実に優先度を下げてください (デフォルトでは +5 nice です)。 iPulse の優先度が低い場合は、ビジー状態では他のアプリケーションに CPU time がより多く与えられます。もう一つの利点は、 CPU が高負荷の時に、iPulse は更新の頻度を自動的に減らし — 他のアプリケーションでのタスクにより多くの CPU time を回します。
  • フローティングウインドウの影を非表示 - iPulse ウインドウに影を表示している場合は、非表示にしてみてください。見栄えはあまりよくありませんが、更新の度に影の位置を再計算する必要がないので CPU time はより少なくて済みます。
  • 情報ウインドウのフェードをオフ - 情報ウインドウのフェードイン/アウトを "なし" に設定してください。ウインドウ表示はより速くなり消費 CPU は少なくなります。

カスタムサウンドの追加 "時報" 機能にカスタムサウンドを追加するには、 AIFF サウンドファイルを ~/Library/Sounds フォルダに追加してください。拡張子が ".aiff" のファイルが iPulse の再起動後にサウンドリストに表示されます。

AIFF サウンドファイルを作成するには、QuickTime Player で MP3, ムービー または 他のメディアのファイルを開きます。ウインドウの底部にあるスライダーを使用してお好みの部分を選択してください。その後 編集 > トリム を使用し ファイル > 書き出し... を選択して選択部分をオーディオファイルに保存します。書き出し時にポップアップメニューで "サウンドから AIFF" が選択されているのを必ず確認してください。

Dock アイコンなしで iPulse を起動する 通常、iPulse は Dock 内にアイコンを表示しています。が、Dock 内にアプリケーションアイコンが表示されない "バックグラウンドモード" として iPulse を起動させるように調整可能です。

アプリケーションをバックグラウンドモードで起動させる場合、幾つかの制限がある点を御承知ください :

  • メニューバーはありません。コマンドキーは動作するのでアプリケーションの終了 (コマンド-Q) や環境設定パネルの表示 (コマンド-コンマ) は可能です。iPulse ウインドウをコントロール-クリックして表示されるコンテキストメニューも使用可能ですのでこれらの機能にはアクセスできます。
  • iPulse はコマンド-オプション-Esc キー使用時に表示される強制終了メニューには表示されません。手動で iPulse のプロセスを終了するにはターミナルウインドウをご使用ください。
  • アプリケーションの設定に変更が必要になりますので、マシン上のすべてのユーザは iPulse をバックグラウンドモードで使用しなければなりません。
バックグラウンドモードに変更する前に、iPulse ウインドウがデスクトップ上に表示されているのを確認してください。

iPulse をバックグラウンドモードで起動させる為の調整を行うには、アプリケーションアイコンを コントロール-クリックして "パッケージの内容を表示" を選択し、アプリケーションパッケージを開いてください。Contents フォルダ内の Info.plist ファイルをテキストエディットで開きます。LSUIElement の項目に移動して "<string>0</string>" を "<string>1</string>" に変更してください。ファイルを保存したら Contents フォルダに戻ってください。

Mac OS 10.2 をお使いの場合は、この時点でiPulse を再起動させてください。10.3 (Panther) をお使いの場合、Info.plist ファイル内に含まれている Launch Services の情報がキャッシュされているので、新規にアプリケーションを作成してキャッシュを更新する必要があります。 Finder 内で iPulse を選択してアプリケーションを複製する (Command+D キーを使用する) のが一番簡単な方法です。それからその新規ファイル (iPulse のコピー) を起動させ、それが Dock 内に表示されていない事を確認してください。この時点でキャッシュは更新されているので、 iPulse を終了して Finder に戻ります。オリジナルの iPulse を削除して、"iPulse のコピー" を "iPulse" に名称変更してください。

バックグラウンドモードで起動させたくない場合は、LSUIElement の値をもとの "0" に戻してください。

パワーユーザ この項目はコマンドラインの使用をためらわない人たちに捧げます。本項では皆さんがターミナルウインドウで快適に操作できるものと仮定して話を進めます。そうでない方は、この項目を読み飛ばす事をオススメします。

環境設定 iPulse 環境設定の大半は環境設定ウインドウ内のパネルで明らかにされています。が、それ以外にも役に立つ設定があります。しかしインタフェースが散らかったり、初心者には理解しにくかったりします。

これらの "パワーユーザ" 向けの環境設定の一つを設定するには、iPulse を終了してコマンドラインで以下の文字列を入力してください : 

% defaults write com.iconfactory.iPulse [preference] [value]
[preference] の部分には環境設定の名称を、[value] の部分には使用する値 (以下のテーブルに示しています) を入力してください。iPulse の再起動後に、新たな環境設定項目を御覧になるはずです。

これらの環境設定を削除してデフォルトの状態に戻すには以下の通り : 

% defaults delete com.iconfactory.iPulse [preference]

さあ、しばしのお別れの後でパワーユーザ用の環境設定が現れます:

環境設定 値 説明
IFApplicationNeverHide 1 数値を 1 に設定すれば、iPulse は隠れません。これで Dock アイコンも半透明にならなくなります。
IFApplicationPlotProcessorArea 0 標準では、iPulse は CPU ゲージをサークル内に表示される領域にパーセント表示で描きます。例えば、使用率 50% ならサークル内の 50% を満たします。見た目にはより正確ですが、CPU 使用率が低ければ値は大きく見えがちです。CPU を関数表示するにはこの値を 0 に設定してください。この場合使用率 50% だとサークルの半分を使って表示されます。
IFApplicationAlwaysHoverTime 0 デフォルトでは、iPulse は日付と時計表示がオフの場合でも時計情報ウインドウを表示します。この数値を 0 に設定すれば、情報ウインドウは日付表示時にのみ表示されます。
IFApplicationHideInfoOnDrag 1 この数値を 1 に設定すれば iPulse フローティングウインドウのドラッグ中は情報ウインドウは隠されます。この設定は旧型のグラフィックカードでは動作しにくく、反応は速くなります。
IFApplicationAlternativeActivity 1 この数値を 1 に設定すればインジケータの表示がゲージのトップに重ならなくなります。
IFApplicationSwapfilesPath /path/to/dir この設定はスワップファイルの位置するパスを特定します。もしスワップファイルの設定をカスタマイズされている場合は、この設定をデフォルトの /var/vm から変更する必要があります。
IFApplicationCHUDWorkaround 1 お使いのマシンに CHUD Tools がインストールされている場合、このフラグを立てる必要があります。さもないと iPulse はカーネルによる CPU time の過剰消費を引き起こします。フラグが 1 の場合は、 iPulse は host_processor_info での PROCESSOR_TEMPERATURE 温度パラメータのコールを回避します。G5 または iBook やアルミボディ PowerBook に搭載の G4 チップなどといった新型プロセッサでは、Mach システムコールの替わりに IOKit 経由で温度が読み込まれるのでこのフラグは無視されます。
IFApplicationInfoDelay 0.75 デフォルトでは、 iPulse の情報ウインドウの表示は (環境設定内のウインドウ項目でチェックがオンになっていれば) 0.75 秒遅れます。この遅延間隔を変えたい場合は、 0 より大きい数値でお好きなものに変更してください。
IFApplicationTrackMouse 0 iPulse は情報ウインドウの情報更新時 (一般環境設定の 時間 タブ内で設定 ) のみ、マウスの位置をチェックします。更新間隔を長くとっている場合、iPulse にマウスの位置をトレースさせればより速いフィードバックを得られます。この機能はより多くプロセッサのリソースを使用する為、デフォルトでは搭載されていません。 — マウスが動くごとに、システム情報を収集する必要があるからです。
IFApplicationStatusAlertThreshold 0.9 メニューバーに表示されるステータス値は 0.0 〜 1.0 の範囲を示しています。このパラメータで定義される範囲を超えた場合、警告色で表示されます。デフォルトでは最大値の 90% に達すると警告色表示となります。
IFApplicationIgnoreExpose 0 If you'd like iPulse to remain on the desktop when you use the Show Desktop feature in Exposé, set this preference to 1. This will make iPulse's transparent windows "sticky" like the Dock and menu bar. This feature uses undocumented calls in the Core Graphics framework, so it may not work in any release other than Panther. But it's pretty cool anyway :-)
